# 🏛️ 팽팽클리닉 마켓 인텔리전스: 확장이 용이한 데이터 구조 설계 (Notion Schema)

본 설계는 추후 **키워드 무한 확장** 및 **플랫폼 추가(신규 성형 앱, 해외 소셜 등)**가 용이하도록 '설정 기반(Config-first)' 구조로 기획되었습니다.

---

## 1. [DB 00] 마켓 인텔리전스 마스터 설정 (Master Config)
이 데이터베이스는 수집할 키워드와 소스를 관리하는 '제어판'입니다.

| 컬럼명 | 속성 | 설명 | 예시 데이터 |
| :--- | :--- | :--- | :--- |
| **모니터링 항목** | Title | 키워드 또는 소스 이름 | #울쎄라볼패임, #강남역실리프팅 |
| **퍼널 단계** | Select | 10단계 중 어디에 속하는가 | 06. 위험 평가, 08. 지역 필터링 |
| **플랫폼** | Multi-select | 어디서 수집하는가 | Naver, Kakao, T-Map, ChatGPT, JP_LIPS |
| **모니터링 주기** | Multi-select | 수집 간격 | Daily, Weekly, Monthly |
| **활성 상태** | Checkbox | 현재 수집 중인지 여부 | Checked (활성) |
| **수집 우선순위** | Select | 데이터 시급성 | P0 (Critical), P1 (High), P2 (Normal) |

---

## 2. [DB 01] Daily Pulse Log (시계열 데이터 저장소)
매일 쌓이는 수치와 텍스트 인사이트를 기록하는 메인 보드입니다.

| 컬럼명 | 속성 | 설명 |
| :--- | :--- | :--- |
| **날짜** | Date | 데이터 수집 시점 |
| **항목 (Relation)** | Relation | [DB 00] 마스터 설정과 연결 |
| **지표 수치** | Number | 검색량, 순위, 포스팅 수 등 |
| **AI 감성 점수** | Formula | 텍스트 분석 결과 (-1.0 ~ 1.0) |
| **핵심 인사이트** | Text | 전문가/AI가 요약한 오늘의 특이사항 |
| **Loss 포착** | Checkbox | 경쟁사 약점이 발견되었는가? |
| **원본 링크** | URL | 원문 확인용 (리뷰, 카페 글 등) |

---

## 3. [DB 02] Competitor Intelligence (경쟁사 10개 추적)
상위 10개 경쟁사(Top 10)의 화력을 정밀 분석합니다.

| 컬럼명 | 속성 | 설명 |
| :--- | :--- | :--- |
| **병원명** | Title | 경쟁 병원 이름 |
| **상태** | Status | 운영 중 / 관찰 대상 / 벤치마킹 완료 |
| **마케팅 채널** | Multi-select | 블로그, 유튜트, 인스타, 틱톡, 홈페이지 |
| **주간 포스팅 수** | Number | 정량적 활동 지표 |
| **평균 별점** | Number | 옴니채널(네이버/카카오/티맵) 통합 별점 |
| **핵심 소구점** | Text | 이 병원이 현재 강조하는 마케팅 포인트 |

---

## 🛠️ 확장을 위한 설계 원칙 (Scalability Rules)

1.  **Tag-based Filtering:** 새로운 시술(예: 티타늄 리프팅)이 나오면 [DB 00]에 키워드 하나만 추가하고 태그만 달아주면, 모든 대시보드 뷰에 자동으로 데이터가 흐르게 설계했습니다.
2.  **Modular Pipeline:** n8n 워크플로우를 소스별(Naver용, Instagram용)로 모듈화하여, 추후 '틱톡'을 추가하고 싶다면 새로운 모듈만 연결하면 됩니다.
3.  **AEO Matrix:** AI(ChatGPT 등)에게 물어본 질문 리스트를 [DB 00]에 등록하여, AI 답변 내 우리 병원의 비중을 매주 자동으로 업데이트합니다.
